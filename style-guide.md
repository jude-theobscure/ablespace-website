# AbleSpace Website Style Guide

---

## ⚠️ CRITICAL RULES (STRICT)

- DO NOT create any illustrations
- DO NOT generate fake UI screenshots
- ONLY use provided real product visuals
- If visuals are missing, use clean placeholders (no AI images)

---

## 🎯 Goal

Create an award-winning, highly interactive SaaS website.

The website must feel:
- Premium
- Smooth
- Modern
- Friendly (not boring SaaS)

---

## 🎨 Visual Style

### Theme
- Light mode only
- Must support future dark mode (clean structure, no hardcoded colors)

### Feel
- Clean + soft + modern
- Slightly friendly but still professional
- Inspired by:
  - Stripe
  - Linear
  - Apple UI

---

## 🟢 Corner Style (VERY IMPORTANT)

Use iOS-style corner smoothing (superellipse feel)

- Target feel: ~60% smoothing
- Soft, organic corners (not sharp)
- Use 16px–24px radius
- Prefer rounded-2xl / rounded-3xl

Corners should feel like Apple UI — soft and fluid

---

## 🧩 Layout Structure (STRICT)

Follow this structure EXACTLY:

1. Navbar
2. Hero Section
3. Product Demo Section
4. Problem Section
5. Features Section
6. Mobile Experience Section
7. AI Features Section
8. Testimonials
9. Schools / District Section
10. Security & Compliance
11. FAQ
12. Final CTA
13. Footer

DO NOT add random sections

---

## 🧠 Product Understanding

Product: AI-powered IEP tracking platform for special education

Key features:
- Data tracking (multiple types)
- Service tracking
- Accommodations
- Reports & graphs (20+)
- Scheduling & billing
- Collaboration
- AI goal generation
- AI worksheets & notes

Pain points:
- Manual data entry
- Missed deadlines
- Poor collaboration
- Data loss risk

Trust:
- HIPAA compliant
- FERPA compliant
- ISO certified
- 20,000+ professionals

---

## ✨ Animations & Interaction (VERY IMPORTANT)

Website must feel alive

Add:
- Smooth scroll animations (fade + slide)
- Staggered animations
- Hover effects (scale, shadow)
- Micro-interactions on buttons
- Sticky navbar
- Smooth transitions

Avoid:
- Heavy animations
- Laggy effects

---

## 📱 Responsiveness (STRICT)

- Mobile-first design
- Works perfectly on:
  - Mobile
  - Tablet
  - Desktop

Mobile:
- Stack layout
- Maintain readability
- Clean spacing

---

## 🎯 UI System

### Spacing
- 8px spacing system

### Components
- Reusable components
- Card-based layout

### Corners
- Rounded (soft, smooth)

### Shadows
- Soft shadows only

---

## 🔤 Typography

- Clean and readable
- Strong hierarchy:
  - H1 → bold
  - H2 → section titles
  - Body → simple

---

## 🎨 Colors

- Light UI
- Soft colors
- No harsh contrast
- White / light gray background

---

## 🚀 Tech Stack

- Next.js
- Tailwind CSS
- Framer Motion

---

## 💡 Design Quality Benchmark

Target:
- Stripe-level polish
- Linear-level smoothness

Avoid:
- Basic template design
- Generic UI

---

## ⚠️ FINAL INSTRUCTION

Focus on EXPERIENCE, not just layout

Make it:
- Premium
- Interactive
- Smooth
